# Ачивки в реальной жизни — дерево достижений (Telegram Mini App)

Технический бриф для разработки. Отдать этот файл и `achievements_data.json` в Claude Code как отправную точку.

## Идея

Telegram Mini App: интерактивное пиксельное дерево. 14 фиксированных ветвей = категории достижений («ветка спорт», «денежная ветка» и т.д.), на каждой ветви растут веточки поменьше — это отдельные достижения. У каждой веточки — пучок листьев: серый, если достижение не выполнено, зелёный — если выполнено. Из ствола, как часть того же дерева, растут корни — это пользовательские достижения, которые каждый добавляет сам.

## Платформа и авторизация

- Только Telegram Mini App (не отдельный сайт в обычном браузере).
- Авторизация через Telegram: приложение получает данные пользователя автоматически из `initData`, которые Telegram передаёт при открытии Mini App внутри бота. Отдельная регистрация, email, пароли — не нужны.
- Подпись `initData` обязательно проверять на сервере (нельзя доверять клиенту), см. ниже про Supabase Edge Function.

## Данные

Файл `achievements_data.json` содержит:
- `branches`: 15 веток, каждая `{id, title}`. Ветка 15 («сделай сам, придумай тоже») — фиксированной структуры не имеет, в неё пользователи добавляют свои пункты через форму.
- `achievements`: 155 фиксированных достижений, каждое `{id, branch_id, order, title, description, requires}`. `requires` — id родительского достижения внутри той же ветки (иерархия вида «получить разряд» → «получить КМС» → «получить мастера спорта»), либо `null`, если достижение базовое и ничего не требует.

Плюс потребуется отдельная таблица в базе для пользовательских «корневых» достижений (id пользователя, текст, дата, статус).

## Визуал дерева

- Дерево рисуется процедурно кодом (не заранее нарисованная картинка), в пиксельном/блочном стиле: ствол по центру, 14 ветвей расходятся от него, корни отходят от основания ствола вниз, визуально являясь частью того же дерева.
- Так как ветви рисует код, у него уже есть точные координаты пути каждой ветви — веточки-достижения автоматически и равномерно распределяются вдоль этого пути, без ручной разметки координат.
- На ветвях разное количество достижений (от 4 до 23) — дерево просто становится больше/гуще в этих местах, искусственно не сжимается.
- Единый размер "пикселя" по всему изображению (рисуем на выровненной пиксельной сетке, без сглаживания/антиалиасинга).
- Навигация: pan & zoom по холсту — можно приблизить конкретную ветвь, можно отдалить и увидеть дерево целиком (кнопка "показать всё").
- Позже эту процедурную отрисовку можно заменить на артом, нарисованным человеком — для этого стоит держать логику раскладки веточек отдельно от конкретной реализации отрисовки ствола/ветвей.

## Взаимодействие

- Наведение на веточку/лист — тултип с названием и описанием достижения.
- Рядом — чекбокс, отмечающий достижение выполненным. Лист сразу перекрашивается серый → зелёный.
- Прогресс сохраняется в базе, персонально для каждого пользователя (по Telegram user id).
- В ветке 15 (корни) — простая форма под/у основания дерева: пользователь текстом добавляет свои достижения, они прорастают как дополнительные корни. Видны только ему самому, другим пользователям — нет.

## Стек

- Frontend: любой SPA-стек (React/Vite и т.п.), собранная как статика.
- Хостинг: GitHub Pages (у пользователя уже есть опыт деплоя туда) — фронтенд полностью статический, обращается к Supabase напрямую из браузера через JS SDK.
- Backend/данные: Supabase — база данных (прогресс пользователей, пользовательские достижения) + Edge Function для проверки подписи `initData` от Telegram (единственная серверная логика, которая нужна).
- Vercel не требуется при таком раскладе.

## Что нужно от пользователя перед стартом (уже собирается им отдельно)

- Токен Telegram-бота (через @BotFather) и адрес задеплоенного сайта, прописанный в настройках Mini App.
- Supabase: URL проекта и anon key.
- Репозиторий на GitHub для деплоя через GitHub Pages.
